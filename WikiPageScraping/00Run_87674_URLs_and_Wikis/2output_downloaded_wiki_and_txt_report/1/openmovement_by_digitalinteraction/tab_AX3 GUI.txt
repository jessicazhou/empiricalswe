<p>The <a class="internal present" href="/digitalinteraction/openmovement/wiki/AX3">AX3</a> OMGUI Configuration and Analysis Tool is an open source, lightweight application designed to provide the following functionality:</p>
<ul>
<li>Setup and configure <a class="internal present" href="/digitalinteraction/openmovement/wiki/AX3">AX3</a> sensors for recording</li>
<li>Download and visualize recorded data from AX3 devices</li>
<li>Provide an easy to use interface to convert binary AX3 recordings</li>
<li>Provide access to validated analysis algorithms on recorded AX3 data</li>
</ul>
<p>This document is intended as a guide to get to grips using the OMGUI software, learn about its features, and serve as a technical reference document.</p>
<hr/>
<h1>
<a aria-hidden="true" class="anchor" href="#downloading-and-installing" id="user-content-downloading-and-installing"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Downloading and Installing</h1>
<p>The OMGUI software is available from the open source GitHub repository:</p>
<blockquote>
<p><a href="https://github.com/digitalinteraction/openmovement/blob/master/Downloads/AX3/AX3-GUI-37.zip?raw=true"><img alt="Download OM GUI" src="https://raw.githubusercontent.com/digitalinteraction/openmovement/master/Docs/omgui/img/download-button.png"/></a> V37 (<a href="https://github.com/digitalinteraction/openmovement/blob/master/Downloads/AX3/AX3-GUI-revisions.md">revisions</a>), Windows, zipped installer.</p>
</blockquote>

<p>The system requirements for the OMGUI software are a Windows PC with XP SP3 or later operating system and Microsoft .NET Framework v3.5 or later.  To install the software, open the downloaded archive and double-click the executable installer package.</p>
<blockquote>
<p><strong>NOTE</strong>: Administrator privileges will be required to install software.  The default location for installation is <code>C:\Program Files</code> but this can be changed to suit user- or system needs.</p>
</blockquote>
<p>To connect with a sensor you will need to install the drivers as well as the software. These drivers come packaged with the OMGUI software and are configured to install as default during the installation process.
The software has an inbuilt facility to update itself for major releases as well as updating the firmware on connected devices.</p>
<p>The OMGUI software uses <code>My Documents</code> as a default working folder location, you can choose any folder to work from.  If you need to share files with other users, consider using a shared or network drive location as the Working Folder.  There is also an <a href="http://www.axivity.com/files/resources/longitudinal_data.zip" rel="nofollow">example data file</a> available to download (place  it in the working folder to run any analysis on it).</p>

<p>This section explains how to configure you device with the default settings and start collecting data. The default settings for configuration are:</p>
<ul>
<li>100Hz logging</li>
<li>Range ±8 g</li>
<li>Start logging on sensor disconnect</li>

<p>This section then proceeds to cover downloading data and exporting to a CSV file with timestamps. For more in-depth instructions and explanations, please read the relevant sections of this manual.</p>

<p>Once configured, click the <strong>OK</strong> button to set the configuration. If you keep the default settings, at this stage the device browser pane will show that the device is set to "Always" record. The device can now be disconnected and it will start recording immediately.</p>

<h2>
<a aria-hidden="true" class="anchor" href="#downloading-the-data" id="user-content-downloading-the-data"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Downloading the data</h2>
<p>After the recording period the next step is to download the data from the device. To accomplish this, plug the device in to the USB port of the computer and wait for it to appear in the <strong>Device Browser Pane</strong>. If the device is still configured to record on disconnect, you will first have to click the <strong>Stop</strong> button. Next, click the Download button in the <strong>Device Toolbar</strong>. The file then start to download to your working folder. Once downloaded the file will appear in the <strong>Local Files Pane</strong>.</p>
<p>The data file is stored in an <strong>.CWA</strong> binary format. This format is not compatible with Excel or other third party software. To get it in a more friendly format it must first be converted using the <strong>Export</strong> function.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#exporting-to-a-comma-separated-value-csv-file" id="user-content-exporting-to-a-comma-separated-value-csv-file"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Exporting to a comma separated value (.CSV) file</h2>
<p>In order to use the raw data with a third-party software package, such as Excel, it must first be converted into a suitable format. This can be done using the <strong>Export Window</strong>. To instigate this, select the file in the <strong>Files Tab</strong> within the Local Files Pane and click <strong>Export</strong> button.</p>
<p>The following window will appear. Set the configuration as shown below and click the OK button to generate the file.</p>
<p><img alt="default settings in CSV Export Window" src="https://raw.githubusercontent.com/digitalinteraction/openmovement/master/Docs/omgui/img/export_dialogue.jpg" title="default settings in CSV Export Window"/></p>
<h2>
<a aria-hidden="true" class="anchor" href="#using-exported-data" id="user-content-using-exported-data"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Using Exported data</h2>
<p>Files generated in using the <strong>Export</strong> function will appear in the <strong>Working Folder</strong>. To open the <strong>Working Folder</strong>, click the button shown below at the side of the <strong>Local Files Pane</strong>.</p>

<p>It is important to note that files exported to CSV format can become very large. Typically software such as Excel can only handle a few hours of data before it struggles. For multi-day recordings, the user should consider working directly with the binary files if developing algorithms, or if the user is happy with one of the existing inbuilt algorithms in the software, using summary measures. These will be covered later in this document.</p>

<h2>
<a aria-hidden="true" class="anchor" href="#configuring-sensors-for-recording" id="user-content-configuring-sensors-for-recording"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Configuring sensors for recording</h2>
<p>The OMGUI software displays all connected devices and their states in the <strong>Device Browser Pane</strong>. This information can be used for checking a devices current configuration as well as making sure the battery level is sufficient to complete the recording required.</p>
<blockquote>
<p><strong>NOTE</strong>: Best practice not to issue a device with less than 85% battery level.</p>
</blockquote>
<p>With a device in the <strong>Device Browser Pane</strong> selected, the buttons on the <strong>Device Toolbar</strong> can be used to perform the following operations:</p>
<ul>
<li>
<p><strong>Download.</strong> If the connected device has any data recordings, this button downloads the recordings to the <strong>Working Folder</strong>.</p>
</li>
<li>
<p><strong>Cancel.</strong> If the connected device is being downloaded, this button terminates the operation.</p>
</li>
<li>
<p><strong>Clear.</strong> If the connected device has and data recordings, this button clears them. This operation is irreversible.</p>
</li>
<li>
<p><strong>Stop.</strong> If the device is currently set to record, this button will clear the configuration so that when the device is disconnected it will enter into <strong>Standby</strong> mode.</p>
</li>
<li>
<p><strong>Identify.</strong> This button will flash the LED on any selected device. It is designed to assist device identification when multiple devices are connected at once.</p>
</li>
<li>
<p><strong>Record.</strong> This button is used to configure the device to record. The button brings up the <strong>Recording Window</strong>:</p>
</li>
</ul>
<p><img alt="Recording Window" src="https://raw.githubusercontent.com/digitalinteraction/openmovement/master/Docs/omgui/img/numbered_recording_dialogue.jpg" title="default settings in CSV Export Window"/></p>
<ol>
<li>
<p><strong>Recording Session ID.</strong> This field provides utility to mark the recording with a unique ID. This is useful in studies where patient data needs to be anonymized as the ID can be linked to a patient record.</p>
</li>
<li>
<p><strong>Sampling Frequency.</strong> This is the number of times per second the sensor data will be recorded. For more information on choosing a sample frequency please see the sensor user manual. The default value is 100Hz which is adequate for most human movement studies.</p>
</li>
<li>
<p><strong>Sampling Range.</strong> This is the maximum value that the sensor will record. For more information on choosing a range please see the sensor manual. The default setting is suitable for most human movement studies.</p>
</li>
<li>
<p><strong>Immediately on Disconnect.</strong> In this configuration the sensor will start recording as soon as it is unplugged from the computer.</p>
</li>
<li>
<p><strong>Interval Start Date/Time.</strong> If the interval option is selected, the user can select a specific date and time to start recording on.</p>
</li>
<li>
<p><strong>Interval Delay.</strong> In interval mode, the user can specify a number of days before the sensor will start recording with the present day as an index. For example “start recording in 2 days time”. The user is also able to directly select a date, and fine tune the start time using the Start Date/Time options.</p>
</li>
<li>
<p><strong>Interval Duration.</strong> In interval mode, the user can specify a duration to record for. If the duration exceeds the physical capabilities of the device, a warning will be given.</p>
</li>
<li>
<p><strong>Interval Stop Date/Time.</strong> In interval mode, the user can specify a specific date and time to stop recording on. If the duration exceeds the physical capabilities of the device, a warning will be given.</p>
</li>
<li>
<p><strong>Study and Subject Meta Data.</strong> The device is able to store meta data along with the recorded data. This section provides the user provision to add a series of descriptive fields that will persist alongside the data recording; thus alleviating the possibility of metadata misalignment at a later date. All fields are optional in this section.</p>
</li>
<li>
<p><strong>Flash during recording.</strong> If checked, this box will put the device in a mode whereby a small flash is periodically given when a device is actually recording (if an interval is set, the device will remain silent during the standby phase). Default is to have the device silently record to avoid wearer disruption.</p>
</li>
<li>
<p><strong>Unpacked data.</strong> If checked, this option will configure the device to adopt a 6-bytes per sample format. Left unchecked (default) the device will adopt a 4 bytes per sample "packed" format, which enables it achieve a logging performance of 14 days at 100Hz with sufficient resolution for human movement studies.</p>
</li>
<li>
<p><strong>Notes Area.</strong> This section is used for warnings to the user about any potential issues with the chosen configuration.</p>
</li>
</ol>
<h2>
<a aria-hidden="true" class="anchor" href="#previewing-and-downloading-data" id="user-content-previewing-and-downloading-data"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Previewing and downloading data</h2>
<p>Any connected devices will appear in the <strong>Device Browser Pane</strong> marked as having data. When highlighted, data on these devices can be previewed in the <strong>Data Preview Window</strong>. There is a <strong>Zoom</strong> tool and <strong>Highlight</strong> tool in the <strong>Selection Tools</strong> section. In the <strong>Preview Filters</strong> section, a number of checkboxes exist to help the user visualize the data. On mousing over the data, the user is able to get a precise data preview at any given instant in the recording.</p>
<p>Recorded data stored on the sensor can be downloaded to the <strong>Working Folder</strong> by clicking the <strong>Download</strong> button. Download time is approximately 6 minutes for 7 days worth of data.</p>
<p>To open and view the contents of the <strong>Working Folder</strong> or change working folders, buttons are available in the <strong>Working Folder Options</strong> part of the interface.</p>

<h3>
<a aria-hidden="true" class="anchor" href="#export-function" id="user-content-export-function"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Export Function</h3>
<p>Captured recordings are stored in a binary format that is not always directly compatible with external program environments. For this reason a built-in export facility is provided to enable captured recordings to be transcribed into various formats.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#export-re-sampled-wav" id="user-content-export-re-sampled-wav"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Export Re-sampled WAV</h3>
<p>This option can be used to transcribe the binary format file into an audio WAVE file (WAV). In this format the data streams are interpolated over the recording duration. This is done via a bi-cubic spline algorithm.
The WAV converter re-samples the binary data file produced on-board the
accelerometer to produces a 4-channel, 16-bit standard WAVE file.  The
first three channels correspond to the X-, Y- and Z-axes from the
accelerometer (and a normalized -1 to 1 value must be scaled by the
range setting, which can be obtained from the WAV header, with a
default of 8 for +/- 8g). The fourth channel contains other
information from the logging device, such as temperature, light and
battery measurements -- encoded into the 16-bit values.</p>
<h3>
<a aria-hidden="true" class="anchor" href="#export-re-sampled-csv" id="user-content-export-re-sampled-csv"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Export Re-sampled CSV</h3>
<p>This operation can be used to convert a binary file into a CSV file format with a fixed sample period; useful for analysis that cannot handle variable sample rates. A bi-cubic interpolation is used to determine timestamps.</p>
<blockquote>
<p><strong>NOTE</strong>: CSV files of samples are considerably larger than binary counterparts and thus consideration to memory requirements must be given when using them.  Binary WAV files should be preferred where it is possible to use them.</p>
</blockquote>

<h2>
<a aria-hidden="true" class="anchor" href="#signal-vector-magnitude-svm" id="user-content-signal-vector-magnitude-svm"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Signal Vector Magnitude (SVM)</h2>
<p>The signal vector magnitude (SVM) has been shown to be a useful measure for working with when axis specific information is not required. The SVM can be calculated in conjunction with, or without, a bandpass filter. Options for using the SVM calculation are now explained:</p>
<p><img alt="SVM calculator window" src="https://raw.githubusercontent.com/digitalinteraction/openmovement/master/Docs/omgui/img/numbered_svm_dialogue.jpg" title="SVM calculator window"/></p>
<ol>
<li>
<p><strong>Epoch.</strong> This is the length of time each SVM averaging is made over. Default is 60 seconds. If number of samples in the file does not fit into an exact integer of epochs, the last epoch will be partial.</p>
</li>
<li>
<p><strong>Mode.</strong> The SVM calculation can be made in a mode where all negative values of the SVM are rectified: abs(sqrt(x^2 + y^2 + z ^2)-1), or discarded: max(0,sqrt(x^2 + y^2 + z ^2)-1)</p>
</li>
</ol>
<p>The output of the SVM tool is a CSV file stored in the working folder.</p>
<p>In detail: the signal vector magnitude takes the accelerometer vector length
from the euclidean distance of the three accelerometer axes, and
subtracts the 1g expected from gravity while stationary:</p>
<pre><code>SVM-1 = sqrt(x^2 + y^2 + z^2) - 1
</code></pre>
<p>This signal can (optionally) be improved by filtering for only
frequencies of human movement, by applying a fourth-order Butterworth
band-pass filter between 0.5Hz and 20Hz. Negative values are dealt
with by either taking the 'absolute' value (default), or clipping them
to zero. This value is aggregated into a mean measurement over
user-defined epoch length (in seconds). The resulting means are then,
either:</p>
<pre><code>sum(abs(sqrt(x^2 + y^2 + z^2) - 1))
sum(max(0, sqrt(x^2 + y^2 + z^2) - 1))
</code></pre>
<p>For more details on the SVM algorithm and the selection of apporpriate cut-points see <a href="http://www.ncbi.nlm.nih.gov/pubmed/21088628" rel="nofollow">1</a>.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#cut-points" id="user-content-cut-points"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Cut-points</h2>
<p>When using the accelerometers to measure physical activity, time spent in specific states has been demonstrated to be a useful output. The <strong>Cut-point</strong> analysis reports the time user has spent in a specific intensity of physical activity where each intensity band is categorized in units of Metabolic Equivalent of Task (METS).</p>
<table>
<thead>
<tr>
<th align="left">Activity Level</th>
<th align="center">MET  Threshold</th>
</tr>
</thead>
<tbody>
<tr>
<td align="left">Sedentry</td>
<td align="center">&lt; 1.5</td>
</tr>
<tr>
<td align="left">Light</td>
<td align="center">1.5 &lt; and &lt; 3.99</td>
</tr>
<tr>
<td align="left">Moderate</td>

</tr>

<blockquote>
<p>1 MET = 3.5 ml O2 . kg^-1 . min ^-1 which is the equivalent of 58.2 W/m^2 or the rate of energy produced per unit surface area of an average person seated at rest.</p>
</blockquote>
<p>To provide utility of converting accelerometry data into METS for specific population groups and for specific mounting sites of the sensor, the following interface is provided:</p>
<p><img alt="cut-points calculator window" src="https://raw.githubusercontent.com/digitalinteraction/openmovement/master/Docs/omgui/img/cutpoints_dialogue.jpg" title="cut-points calculator window"/></p>
<p>This algorithm is based on that proposed in <a href="http://journals.plos.org/plosone/article?id=10.1371/journal.pone.0022922" rel="nofollow">2</a>, which takes an abs(SVM-1) values at 80Hz summed over 60 seconds.  Our implementation compares the mean SVM over 60 seconds against 'cut points' which divide the results into time spent in activity levels: sedentary (&lt; 1.5 METS), light (&gt;= 1.5 METS, &lt; 4
METS), moderate (&gt;= 4 METS, &lt; 7 METS), and vigorous (&gt;=7 METS). The results are summed over user-defined multiples of the 1-minute epoch.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#wear-time-validation-wtv" id="user-content-wear-time-validation-wtv"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Wear Time Validation (WTV)</h2>
<p>The wear time validation (WTV) algorithm is designed to give a high level view if the device has been worn or not. It is unspecific to mounting site and based on the accelerometer readings. A screen-shot of the interface is give below.</p>
<p><img alt="WTV calculator window" src="https://raw.githubusercontent.com/digitalinteraction/openmovement/master/Docs/omgui/img/wtv_dialogue.jpg" title="WTV calculator window"/></p>
<p>The algorithm is based on the method proposed in <a href="http://journals.plos.org/plosone/article?id=10.1371/journal.pone.0022922" rel="nofollow">2</a>, where accelerometer non-wear time is estimated from the standard deviation and range of each accelerometer axis calculated over 30-minute periods. Each period is classified as non-wear time if the standard deviation was less than
3.0 mg for at least two out of the three axes, or if the value range,
for at least two out of three axes, was less than 50 mg.  The results
are summed for user-defined multiples of 30 minute intervals.</p>
<p>The units reported in the WTV are the sum of the number of 30min windows detected within the epoch period. For example, if the epoch period is 1 hour and the sensor is worn continuously, the result will be 2. If the sensor is worn for the first 30 mins and then removed, the result will be 1. In the WTV process, windows are non-overlapping and treated as discrete chunks.</p>
<h2>
<a aria-hidden="true" class="anchor" href="#sleep" id="user-content-sleep"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z" fill-rule="evenodd"></path></svg></a>Sleep</h2>

"Towards a Benchmark for Wearable Sleep Analysis with Inertial Wrist-worn Sensing Units",
